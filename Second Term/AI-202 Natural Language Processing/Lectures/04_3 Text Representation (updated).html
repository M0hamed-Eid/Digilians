01
Introduction to Text Representation
▼

The core problem

Computers are extremely fast at processing numbers. But language — words, sentences, documents — is not numbers. So before any NLP model can work on text, we must first convert that text into numbers. This process is called Text Representation.

Analogy
Think of a computer as a calculator that only understands digits. If you want it to "read" the word "apple", you need to translate "apple" into a number or a set of numbers first. Text representation is exactly that translation step.

Why does it matter?

  • Every ML and deep learning model needs numerical input — no exceptions.
  • The way you represent text directly affects how well your model learns. A bad representation loses information; a good one preserves meaning.
  • Text representation is also called feature engineering — turning raw text into useful features.

The spectrum of methods

Methods range from very simple to very sophisticated:

MethodIdeaCaptures meaning?
One-Hot EncodingEach word = a unique binary vectorNo
Bag of Words / Count VectorizerCount word occurrencesPartially
TF-IDFWeight words by importanceBetter
N-GramCount word group occurrencesPartially (context)
Co-Occurrence Matrix / Word EmbeddingsWords with similar context = similar vectorsYes
Summary
Text Representation = converting text into numbers so machines can process it. It is the essential first step for all NLP tasks. Methods range from simple (one-hot) to advanced (embeddings). The goal is always to preserve as much meaning as possible.

المشكلة الأساسية

الحاسوب سريع جداً في معالجة الأرقام. لكن اللغة — كلمات، جمل، وثائق — ليست أرقاماً. لذا قبل أي model في NLP يقدر يشتغل على النص، لازم نحوّل هذا النص لأرقام أولاً. هذه العملية اسمها Text Representation.

تشبيه
فكّر في الحاسوب كآلة حاسبة تفهم الأرقام فقط. لو تبي إياها "تقرأ" كلمة "apple"، لازم تترجم "apple" لرقم أو مجموعة أرقام أولاً. الـ Text Representation هي بالضبط خطوة الترجمة هذه.

ليش مهم؟

  • كل model في ML وDeep Learning يحتاج مدخلات رقمية — بدون استثناء.
  • طريقة تمثيلك للنص بتأثر مباشرة على مدى تعلّم الـ model. تمثيل سيء يفقد المعلومات؛ تمثيل جيد يحافظ على المعنى.
  • الـ Text Representation تُعرف أيضاً بـ feature engineering — تحويل النص الخام لـ features مفيدة.

02
One-Hot Encoding
▼

What is One-Hot Encoding?

Given a vocabulary (the complete list of unique words), each word is represented as a vector of length N (where N = total number of words in the vocabulary). The vector contains all zeros except for a single 1 in the position that belongs to that word.

Analogy
Imagine a class with 5 seats. Each student "is" represented by the seat they sit in: student A = [1,0,0,0,0], student B = [0,1,0,0,0], etc. No student shares a representation, and no representation tells you anything about who sits near whom.

Visual example

Vocabulary: ["cat", "dog", "fish", "bird"] — 4 words, so each vector has length 4.

WordVector
cat[1, 0, 0, 0]
dog[0, 1, 0, 0]
fish[0, 0, 1, 0]
bird[0, 0, 0, 1]

Limitations — why it fails at scale

1. The curse of dimensionality

A real vocabulary has 50,000+ words. Every word's vector has 49,999 zeros and a single 1. To store all word vectors you need 50,000 × 50,000 = 2.5 billion memory units. This is called a sparse representation — mostly zeros, very little signal.

2. No meaning or similarity

The words "orange", "banana", and "watermelon" are all fruits. In our mental map they are close together. But in one-hot space, every pair of words is exactly the same distance apart. "cat" and "dog" are just as "different" from each other as "cat" and "spaceship". No semantic relationships are captured.

3. No frequency, no order, no context

  • Does not count how often a word appears.
  • Does not use the order of words in a sentence.
  • Each word is encoded in complete isolation.
Key Takeaway
One-hot encoding is the simplest possible representation. It is useful for understanding the concept of word vectors, but is never used in real NLP tasks due to sparsity and the lack of any semantic information.
Summary
One-hot encodes each word as a vector of zeros with a single 1. Simple and clear, but fails in practice: huge memory cost (sparse), treats all words as equally different, and captures zero meaning or context.

ما هو One-Hot Encoding؟

بناءً على vocabulary (القائمة الكاملة للكلمات الفريدة)، كل كلمة تُمثَّل كـ vector طوله N (حيث N = إجمالي عدد الكلمات في الـ vocabulary). الـ vector يحتوي كل أصفار ما عدا رقم 1 واحد في موضع تلك الكلمة.

تشبيه
تخيّل فصل بـ 5 مقاعد. كل طالب "يُمثَّل" بالمقعد اللي يجلس فيه: الطالب A = [1,0,0,0,0]، الطالب B = [0,1,0,0,0]، إلخ. لا طالب يشارك تمثيل طالب آخر، ولا تمثيل يخبرك شيئاً عمّن يجلس بجانب من.

مثال بصري

الـ Vocabulary: ["cat", "dog", "fish", "bird"] — 4 كلمات، إذن كل vector طوله 4.

الكلمةالـ Vector
cat[1, 0, 0, 0]
dog[0, 1, 0, 0]
fish[0, 0, 1, 0]
bird[0, 0, 0, 1]

المحدودية — ليش بتفشل مع الحجم الكبير

1. لعنة الأبعاد (Curse of Dimensionality)

في الواقع الـ vocabulary يحتوي 50,000+ كلمة. كل vector يحتوي 49,999 صفر ورقم 1 واحد. لتخزين كل vectors الكلمات تحتاج 50,000 × 50,000 = 2.5 مليار وحدة ذاكرة. هذا ما يسمى تمثيل sparse — أغلبه أصفار وفيه إشارة قليلة جداً.

2. لا معنى ولا تشابه

كلمات "orange"، "banana"، "watermelon" كلها فواكه — في خريطتنا الذهنية هي قريبة بعضها من بعض. لكن في فضاء One-Hot، كل زوج من الكلمات على نفس المسافة تماماً من بعض. "cat" و"dog" بنفس قدر "اختلاف" "cat" عن "spaceship". لا علاقات دلالية تُحتجز.

3. لا تكرار ولا ترتيب ولا سياق

  • لا يعدّ كم مرة ظهرت الكلمة.
  • لا يستخدم ترتيب الكلمات في الجملة.
  • كل كلمة مشفّرة بعزل تام عن باقي الكلمات.
نقطة مهمة
One-Hot هو أبسط تمثيل ممكن. مفيد لفهم مفهوم word vectors، لكن لا يُستخدم في مهام NLP الحقيقية بسبب الـ sparsity وغياب أي معلومات دلالية.
ملخص
One-Hot يشفّر كل كلمة كـ vector من أصفار مع رقم 1 واحد. بسيط وواضح، لكن يفشل عملياً: تكلفة ذاكرة ضخمة (sparse)، يعامل كل الكلمات كأنها متساوية في الاختلاف، ولا يحتجز أي معنى أو سياق.
03
Bag of Words (BoW)
▼

What is Bag of Words?

Bag of Words (BoW) represents a document by counting how many times each word in the vocabulary appears in it. It literally treats a document as a bag — you shake the bag, the words fall out, and you just count each type. The order is gone.

Analogy
Imagine tearing a book apart, throwing all the words into a bag, and shaking it. You then count: 50 "the"s, 30 "king"s, 20 "sword"s... The count profile represents the book. You have no idea what order the words were in, but the counts give you a rough sense of the topic.

Two building blocks

  • A vocabulary of known words (built from your corpus).
  • A count measure — how many times each vocabulary word appears in a given document.
Example

Corpus (2 documents):

  • Doc 1: "the cat sat on the mat"
  • Doc 2: "the cat ate the rat"

Vocabulary: [the, cat, sat, on, mat, ate, rat]

Documentthecatsatonmataterat
Doc 12111100
Doc 22100011

Each row is the document's BoW vector. Doc 1 = [2,1,1,1,1,0,0]

Limitations

ProblemExplanation
Vocabulary designNeeds careful preprocessing (stop word removal, stemming). A huge vocabulary = huge sparse vectors.
SparsityMost documents use only a tiny fraction of the vocabulary. Most counts are zero — computationally expensive.
No word order / no meaning"This is interesting" and "Is this interesting?" produce the same BoW. Synonyms like "old bike" vs "used bike" are treated as different.
Summary
BoW counts word occurrences in a document, ignoring all word order. Simple and effective for basic tasks. Main problems: sparsity, no semantic understanding, and no awareness of word order or context.

ما هو Bag of Words؟

Bag of Words (BoW) يمثّل وثيقة بعدّ كم مرة ظهرت كل كلمة من الـ vocabulary فيها. حرفياً يعامل الوثيقة كحقيبة — تهزّها، الكلمات تسقط، وتعدّ كل نوع. الترتيب يضيع.

تشبيه
تخيّل إنك تمزّق كتاباً، ترمي كل الكلمات في حقيبة، وتهزّها. ثم تعدّ: 50 مرة "the"، 30 مرة "king"، 20 مرة "sword"... ملف العدّ هذا يمثّل الكتاب. ما تعرف ترتيب الكلمات، لكن الأعداد تعطيك فكرة تقريبية عن الموضوع.

المكوّنان الأساسيان

  • vocabulary من الكلمات المعروفة (مبني من الـ corpus).
  • مقياس العدّ — كم مرة ظهرت كل كلمة من الـ vocabulary في وثيقة معينة.
مثال

Corpus (وثيقتان):

  • Doc 1: "the cat sat on the mat"
  • Doc 2: "the cat ate the rat"

Vocabulary: [the, cat, sat, on, mat, ate, rat]

الوثيقةthecatsatonmataterat
Doc 12111100
Doc 22100011

Doc 1 = [2,1,1,1,1,0,0]

المحدودية

المشكلةالشرح

Sparsityمعظم الوثائق تستخدم جزءاً صغيراً فقط من الـ vocabulary. معظم الأعداد أصفار.

ملخص
BoW يعدّ تكرار الكلمات في وثيقة، متجاهلاً ترتيبها. بسيط وفعّال للمهام الأساسية. المشاكل الرئيسية: sparsity، لا فهم دلالي، ولا وعي بترتيب الكلمات أو سياقها.
04
Count Vectorizer
▼

What is Count Vectorizer?

Count Vectorizer is the practical implementation of Bag of Words. It squeezes an entire sentence or document into a single vector. Each position in the vector corresponds to a specific token from the vocabulary, and the value is how many times that token appeared.

Analogy
Think of a spreadsheet where columns are vocabulary words and rows are documents. Count Vectorizer fills each cell with the word count. It is a structured, numeric summary of a text — like a fingerprint that captures word frequency but not word meaning or order.

How it works step by step

  1. Build the vocabulary: scan all documents, collect unique tokens, assign each an ID.
  2. Vectorize each document: for each document, create a vector of length = vocabulary size. Set each position to the count of that token in the document.
Code Example (scikit-learn)
from sklearn.feature_extraction.text import CountVectorizer corpus = [ "I love NLP and I love Python", "Python is great for NLP", ] cv = CountVectorizer() X = cv.fit_transform(corpus) print(cv.get_feature_names_out()) # ['and', 'for', 'great', 'is', 'love', 'nlp', 'python'] print(X.toarray()) # Doc 1: [1, 0, 0, 0, 2, 1, 1] ← "love" appears 2x # Doc 2: [0, 1, 1, 1, 0, 1, 1]

Hashing Vectorizer — memory-efficient variant

Instead of storing the vocabulary as a string dictionary, a Hashing Vectorizer applies a hash function to each token to convert it directly to a numerical index. This uses much less memory (no vocabulary object stored), but you lose the ability to see which word maps to which index.

Summary
Count Vectorizer is the practical BoW implementation in scikit-learn. It builds a vocabulary from the corpus, then represents each document as a word-count vector. The Hashing Vectorizer is a memory-efficient alternative that uses hash functions instead of a stored vocabulary.

ما هو Count Vectorizer؟

Count Vectorizer هو التطبيق العملي لـ Bag of Words. يضغط جملة أو وثيقة كاملة في vector واحد. كل موضع في الـ vector يقابل token محدد من الـ vocabulary، والقيمة هي عدد مرات ظهوره.

تشبيه
فكّر في جدول بيانات (spreadsheet) حيث الأعمدة هي كلمات الـ vocabulary والصفوف هي الوثائق. Count Vectorizer يملأ كل خلية بعدد الكلمة. إنه ملخص رقمي منظم للنص — مثل بصمة تلتقط تكرار الكلمات لكن لا معناها ولا ترتيبها.

كيف يعمل خطوة بخطوة

  1. بناء الـ vocabulary: فحص كل الوثائق، جمع الـ tokens الفريدة، إعطاء كل منها ID.
  2. تحويل كل وثيقة لـ vector: لكل وثيقة، إنشاء vector بطول = حجم الـ vocabulary. وضع عدد كل token في موضعه.
مثال كود (scikit-learn)
from sklearn.feature_extraction.text import CountVectorizer corpus = [ "I love NLP and I love Python", "Python is great for NLP", ] cv = CountVectorizer() X = cv.fit_transform(corpus) print(cv.get_feature_names_out()) # ['and', 'for', 'great', 'is', 'love', 'nlp', 'python'] print(X.toarray()) # Doc 1: [1, 0, 0, 0, 2, 1, 1] ← "love" ظهرت مرتين # Doc 2: [0, 1, 1, 1, 0, 1, 1]

Hashing Vectorizer — البديل الموفّر للذاكرة

بدل تخزين الـ vocabulary كـ dictionary نصي، يطبّق Hashing Vectorizer hash function على كل token لتحويله مباشرة لـ index رقمي. يستخدم ذاكرة أقل بكثير (لا كائن vocabulary يُخزَّن)، لكن تفقد القدرة على رؤية أي كلمة يقابلها أي index.

ملخص
Count Vectorizer هو التطبيق العملي لـ BoW في scikit-learn. يبني vocabulary من الـ corpus ثم يمثّل كل وثيقة كـ vector عدّ الكلمات. الـ Hashing Vectorizer بديل موفّر للذاكرة يستخدم hash functions بدل vocabulary مخزون.
05
TF-IDF
▼

The problem Count Vectorizer misses

Count Vectorizer gives the same weight to every word. But the word "the" appears in almost every document — it tells you nothing useful. Meanwhile "blockchain" appearing 5 times in a document is a much stronger signal. We need a smarter weighting system.

TF-IDF: two ideas multiplied together

TF-IDF stands for Term Frequency × Inverse Document Frequency. It rewards words that appear often in a specific document but rarely across all documents.

Analogy
Imagine you are grading how "special" a word is to a document. A word is special if: (1) it appears a lot in this document AND (2) it does not appear in every other document. If it appears everywhere, it is boring and not useful. If it appears rarely everywhere but a lot here, it is the keyword of this document.

The two formulas

TF(t, d) = (count of term t in document d) / (total words in document d)
IDF(t) = log( total documents / documents containing t )
TF-IDF(t, d) = TF(t, d) × IDF(t)

Step-by-step worked example

Example from slides

Setup: a document of 100 words where "apple" appears 5 times. We have 10 million documents total, and "apple" appears in 1,000 of them.

Step 1 — TF:

TF = 5 / 100 = 0.05

Step 2 — IDF:

IDF = log(10,000,000 / 1,000) = log(10,000) = 4

Step 3 — TF-IDF:

TF-IDF = 0.05 × 4 = 0.20

The word "apple" gets a weight of 0.20 in this document. A common word like "the" would get IDF ≈ 0 (appears in almost all documents), so its TF-IDF ≈ 0.

What a high TF-IDF score means

TFIDFTF-IDF scoreMeaning
HighHighHighWord is important to this specific document
HighLowLowWord is common everywhere (like "the")
LowHighLowWord rarely appears anywhere, including here
Code Example (scikit-learn)
from sklearn.feature_extraction.text import TfidfVectorizer corpus = [ "I love NLP and machine learning", "Machine learning is powerful", "NLP is a part of machine learning", ] tfidf = TfidfVectorizer() X = tfidf.fit_transform(corpus) print(tfidf.get_feature_names_out()) print(X.toarray().round(2)) # Words common to all docs get lower scores # Unique words like "love" get higher scores in their document
Summary
TF-IDF = TF × IDF. TF rewards words that appear often in a document. IDF penalizes words that appear in every document. The result: common words (the, is) get near-zero weight; rare, document-specific words get high weight. Much more informative than raw counts.

المشكلة التي يفوّتها Count Vectorizer

الـ Count Vectorizer يعطي نفس الوزن لكل كلمة. لكن كلمة "the" تظهر في تقريباً كل وثيقة — لا تخبرك شيئاً مفيداً. بينما ظهور "blockchain" 5 مرات في وثيقة هو إشارة أقوى بكثير. نحتاج نظام وزن أذكى.

TF-IDF: فكرتان مضروبتان ببعض

TF-IDF اختصار لـ Term Frequency × Inverse Document Frequency. يكافئ الكلمات التي تظهر كثيراً في وثيقة محددة لكن نادراً عبر كل الوثائق.

تشبيه
تخيّل إنك بتقيّم مدى "تميّز" كلمة لوثيقة ما. كلمة متميزة إذا: (1) ظهرت كثيراً في هذه الوثيقة، و(2) لا تظهر في كل وثيقة أخرى. لو ظهرت في كل مكان فهي مملة وغير مفيدة. لو ظهرت نادراً في أي مكان لكن كثيراً هنا، فهي الكلمة المفتاحية لهذه الوثيقة.

الصيغتان الرياضيتان

TF(t, d) = (عدد مرات ظهور t في d) / (إجمالي كلمات d)
IDF(t) = log( إجمالي الوثائق / الوثائق التي تحتوي t )
TF-IDF(t, d) = TF(t, d) × IDF(t)

مثال محلول خطوة بخطوة

مثال من الشرائح

الإعداد: وثيقة من 100 كلمة، فيها كلمة "apple" تظهر 5 مرات. لدينا 10 ملايين وثيقة، و"apple" تظهر في 1,000 منها.

الخطوة 1 — TF:

TF = 5 / 100 = 0.05

الخطوة 2 — IDF:

IDF = log(10,000,000 / 1,000) = log(10,000) = 4

الخطوة 3 — TF-IDF:

TF-IDF = 0.05 × 4 = 0.20

كلمة "apple" تحصل على وزن 0.20 في هذه الوثيقة. كلمة شائعة مثل "the" ستحصل على IDF ≈ 0 (تظهر في تقريباً كل الوثائق)، فـ TF-IDF ≈ 0.

ماذا يعني درجة TF-IDF عالية

TFIDFدرجة TF-IDFالمعنى
عاليعاليعاليةالكلمة مهمة لهذه الوثيقة تحديداً
عاليمنخفضمنخفضةكلمة شائعة في كل مكان (مثل "the")
منخفضعاليمنخفضةكلمة نادرة في كل مكان بما فيه هنا
ملخص
TF-IDF = TF × IDF. TF يكافئ الكلمات الكثيرة في وثيقة. IDF يعاقب الكلمات الموجودة في كل وثيقة. النتيجة: كلمات شائعة (the, is) تحصل على وزن صفري تقريباً؛ كلمات نادرة ومرتبطة بالوثيقة تحصل على وزن عالٍ. أكثر إفادة بكثير من الأعداد الخام.
06
N-Gram
▼

What is N-Gram?

N-Gram is an extension of Bag of Words where instead of counting individual words, we count groups of N consecutive words. This gives the model some awareness of word sequences and captures short-range context that BoW misses.

Analogy
BoW hears each instrument in an orchestra separately: "violin... drums... flute...". N-Gram (specifically bigram) hears pairs: "violin-drums, drums-flute, flute-piano...". The pairs give you more context about how the music flows than isolated instruments do.

Types of N-Grams

NameN valueExample from "I love deep learning"
Unigram1I, love, deep, learning
Bigram2"I love", "love deep", "deep learning"
Trigram3"I love deep", "love deep learning"
Example

Sentence: "New York is a great city"

Unigrams: New, York, is, a, great, city

Bigrams: New York, York is, is a, a great, great city

With bigrams, "New York" is treated as a single unit — preserving the place name. With unigrams, "New" and "York" are independent, losing the meaning.

Code Example (scikit-learn)
from sklearn.feature_extraction.text import CountVectorizer corpus = ["I love deep learning", "deep learning is powerful"] # Bigrams only cv_bigram = CountVectorizer(ngram_range=(2, 2)) X = cv_bigram.fit_transform(corpus) print(cv_bigram.get_feature_names_out()) # ['deep learning', 'is powerful', 'learning is', 'love deep'] # Unigrams + Bigrams together cv_both = CountVectorizer(ngram_range=(1, 2))

Trade-off: more context vs more dimensions

N-Grams capture more context, but the vocabulary explodes: a vocabulary of 50,000 words has ~50,000² possible bigrams. In practice, only common bigrams are kept (using min_df and max_features parameters).

Summary
N-Gram counts groups of N consecutive words. Bigrams (N=2) and trigrams (N=3) capture short-range context that BoW misses (e.g., "New York" as a unit). The trade-off: bigger vocabulary, more dimensions, more memory.

ما هو N-Gram؟

N-Gram هو امتداد لـ Bag of Words حيث بدل عدّ الكلمات المنفردة، نعدّ مجموعات من N كلمات متتالية. يعطي الـ model وعياً ببعض تسلسل الكلمات ويلتقط سياقاً قصير المدى يفوّته BoW.

تشبيه
BoW يسمع كل آلة موسيقية في الأوركسترا منفصلة: "كمان... طبول... ناي...". N-Gram (تحديداً bigram) يسمع ثنائيات: "كمان-طبول، طبول-ناي، ناي-بيانو...". الثنائيات تعطيك سياقاً أكبر عن تدفق الموسيقى من الآلات المنفصلة.

Occurrence Matrix & Co-Occurrence Matrix
▼

The key idea: distributional semantics

These methods are based on a famous linguistic hypothesis: "A word can be identified by the company it keeps." Words that tend to appear in the same contexts have similar meanings. "Doctor" and "nurse" often appear near "hospital", "patient", "medicine" — so they are semantically close.

Occurrence Matrix (Term-Document Matrix)

A rectangular matrix where:

  • Rows = unique words (terms) — the vocabulary
  • Columns = documents
  • Each cell = 0/1, raw frequency, or TF-IDF score
Example — Occurrence Matrix

Corpus: "I like deep learning.", "I like NLP.", "I enjoy flying."

This is also called a Term-Document Matrix. Two rows that look similar = two terms that appear in similar documents = potentially similar meaning.

Co-Occurrence Matrix (Term-Term Matrix)

A square matrix where both axes are the vocabulary. Each cell counts how many times two terms appeared near each other (within a sliding context window).

Example — Co-Occurrence with Window=1

Corpus: "I like deep learning.", "I like NLP.", "I enjoy flying."

"I" co-occurs with "like" twice (in Doc 1 and Doc 2) and with "enjoy" once (in Doc 3). Notice it is a symmetric matrix — if A appears near B, then B appears near A.

Two ways to define "context"

ApproachDefinitionCaptures
Full DocumentTwo words co-occur if they are in the same documentBroad topical similarity
Window-basedTwo words co-occur if they are within N words of each other (window size 3, 5, 7, 9 are common)Syntactic + semantic similarity

Limitations of Co-Occurrence Matrix

  • Size grows with vocabulary squared — very large for real vocabularies.
  • Highly sparse — most word pairs never co-occur.
  • Needs dimensionality reduction (e.g., SVD — Singular Value Decomposition) to be practical.
  • Hard to update when new words or documents are added.
Bridge to Word Embeddings
The co-occurrence matrix is the conceptual foundation for modern word embeddings like Word2Vec and GloVe. Those methods learn dense, low-dimensional vectors that capture the same "words near each other = similar meaning" idea — but far more efficiently than storing the full matrix.
Summary
Occurrence Matrix = terms × documents (rectangular). Co-Occurrence Matrix = terms × terms (square), counting how often two words appear near each other. Both are based on the idea that word meaning comes from context. Main limitation: huge, sparse, needs SVD for practical use. These lead directly to Word2Vec and modern embeddings.

الفكرة الأساسية: الدلالة التوزيعية (Distributional Semantics)

هذه الأساليب مبنية على فرضية لغوية مشهورة: "الكلمة تُعرَّف بالصحبة التي تحتفظ بها." الكلمات التي تميل للظهور في نفس السياقات لها معاني متشابهة. "Doctor" و"nurse" كثيراً ما تظهران بجانب "hospital"، "patient"، "medicine" — فهما متقاربتان دلالياً.

Occurrence Matrix (Term-Document Matrix)

مصفوفة مستطيلة حيث:

  • الصفوف = الكلمات الفريدة (terms) — الـ vocabulary
  • الأعمدة = الوثائق
  • كل خلية = 0/1، أو تكرار خام، أو درجة TF-IDF

صفّان متشابهان = كلمتان تظهران في وثائق مشابهة = ربما لهما معنى مشابه.

Co-Occurrence Matrix (Term-Term Matrix)

مصفوفة مربعة حيث كلا المحورين هو الـ vocabulary. كل خلية تعدّ كم مرة ظهر كلا الـ terms بالقرب من بعضهما (ضمن نافذة سياق متحركة).

مثال — Co-Occurrence بنافذة = 1

Corpus: "I like deep learning.", "I like NLP.", "I enjoy flying."

"I" تتزامن مع "like" مرتين (في Doc 1 وDoc 2) ومع "enjoy" مرة واحدة (في Doc 3). المصفوفة متماثلة (symmetric) — لو A ظهر بجانب B، فـ B ظهر بجانب A.

محدودية Co-Occurrence Matrix

  • الحجم يكبر مع مربع الـ vocabulary — ضخم جداً للـ vocabularies الحقيقية.
  • Sparse جداً — معظم أزواج الكلمات لا تتزامن أبداً.
  • يحتاج تقليص الأبعاد (مثل SVD — Singular Value Decomposition) ليكون عملياً.
  • صعب التحديث عند إضافة كلمات أو وثائق جديدة.
الجسر نحو Word Embeddings
الـ Co-Occurrence Matrix هو الأساس المفاهيمي للـ word embeddings الحديثة مثل Word2Vec وGloVe. تلك الطرق تتعلم vectors كثيفة (dense) وقليلة الأبعاد تلتقط نفس فكرة "الكلمات القريبة = معنى متشابه" — لكن بكفاءة أعلى بكثير من تخزين المصفوفة الكاملة.
ملخص
Occurrence Matrix = terms × documents (مستطيلة). Co-Occurrence Matrix = terms × terms (مربعة)، تعدّ كم مرة ظهرت كلمتان بالقرب من بعضهما. كلتاهما مبنيتان على فكرة أن معنى الكلمة يأتي من سياقها. المحدودية الرئيسية: ضخمة، sparse، تحتاج SVD للاستخدام العملي. هذا يقودنا مباشرة إلى Word2Vec والـ embeddings الحديثة.
