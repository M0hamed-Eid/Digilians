01
Introduction to Text Pre-Processing
▼

What is Text Pre-Processing?

Before you can use a machine learning model on text, the text needs to be cleaned and standardized. This is called text pre-processing.

Think of raw text as messy, unstructured data — it has noise, inconsistency, and unnecessary parts. Pre-processing removes all of that and converts it into a form that a machine can work with.

Analogy
Imagine you are cooking. Before you cook vegetables, you wash them, peel them, and cut them into pieces. You do not throw the whole dirty vegetable into the pan. Text pre-processing is exactly this "prepare before cooking" step for text data.

Why does it matter?

  • Machines do not understand language the way humans do — they need numbers and patterns.
  • Raw text is noisy — it has uppercase, punctuation, meaningless words, and different forms of the same word.
  • Pre-processing reduces noise and makes the model's job easier and more accurate.

Important note

Not every pre-processing step is needed for every task. You choose the steps based on your specific use case. The order of steps also matters.

Summary
Text pre-processing = cleaning and preparing raw text before feeding it to any NLP model. It improves model accuracy and reduces noise. The steps you choose depend on your task.

ما هو Text Pre-Processing؟

قبل ما تقدر تستخدم أي model على النص، لازم تنظّف النص وتجهّزه. هذه العملية اسمها Text Pre-Processing أو "المعالجة المسبقة للنص".

فكّر في النص الخام (raw text) على أنه بيانات فوضوية — فيها ضوضاء (noise)، وكلمات زيادة، وأشكال مختلفة لنفس الكلمة. المعالجة المسبقة بتشيل كل هذا وتحوّله لشكل تقدر الماكينة تفهمه.

تشبيه
تخيّل إنك بتطبخ خضار. قبل الطبخ، بتغسله وبتقشّره وبتقطّعه. ما بترمي الخضرة كاملة وسخة في الطنجرة. Pre-Processing هي نفس هذه الخطوة — تجهيز المكوّنات قبل الطبخ.

ليش مهم؟

  • الماكينة ما بتفهم اللغة زي الإنسان — هي بتحتاج أرقام وأنماط.
  • النص الخام فيه حروف كبيرة وصغيرة، علامات ترقيم، وكلمات غير مفيدة.
  • المعالجة المسبقة بتقلّل الضوضاء وبتخلي الـ model يشتغل بدقة أكبر.

ملاحظة مهمة

مش كل خطوة لازم تستخدمها في كل مشروع. بتختار الخطوات حسب المهمة اللي عندك، والترتيب كمان مهم.

ملخص
Text Pre-Processing = تنظيف وتجهيز النص الخام قبل إدخاله لأي model. بيحسّن الدقة ويقلّل الضوضاء. الخطوات اللي تختارها بتعتمد على مهمتك.
02
Regular Expressions (Regex)
▼

What is a Regular Expression?

A Regular Expression (Regex) is a sequence of characters that defines a search pattern. Think of it as a powerful "find" tool for text — much stronger than Ctrl+F.

Analogy
Imagine you want to find all phone numbers in a 1000-page document. You could read all 1000 pages, or you could say "find any pattern that looks like: 3 digits, dash, 3 digits, dash, 4 digits." Regex is that second approach — describe the pattern, find all matches automatically.

Main building blocks

TypeWhat it doesExample
MetacharactersCharacters with special meaning. matches any character, ^ means start of line
Special sequencesBackslash + letter = shortcut\d = any digit, \w = any word char
Sets [ ]Match any char inside brackets[aeiou] matches any vowel

Main Regex functions in Python (re module)

  • findall() — finds ALL matches, returns a list
  • search() — finds the FIRST match only, returns a match object
  • split() — splits the string at each match
  • sub() — replaces matches with new text
Code Example
import re text = "My email is john@example.com and work is john@corp.org" # Find all emails emails = re.findall(r'\b\w+@\w+\.\w+\b', text) # Result: ['john@example.com', 'john@corp.org'] # Replace emails with [HIDDEN] cleaned = re.sub(r'\b\w+@\w+\.\w+\b', '[HIDDEN]', text) # Result: 'My email is [HIDDEN] and work is [HIDDEN]'

Use cases in NLP

  • Cleaning text: removing HTML tags, URLs, special characters
  • Tokenization using custom patterns
  • Validating input (emails, phone numbers)
  • Extracting structured info from unstructured text
Summary
Regex is a pattern-matching tool. In NLP, it is mainly used for cleaning text and basic tokenization. Python's re module gives you findall, search, split, and sub.

ما هي Regular Expressions؟

Regular Expression (Regex) هي تسلسل من الحروف بيعرّف نمط بحث (search pattern). فكّر فيها كأداة "Find" قوية جداً — أقوى بكثير من Ctrl+F العادي.

تشبيه
تخيّل إنك تبي تلاقي كل أرقام الهاتف في وثيقة من 1000 صفحة. ممكن تقرأ كل الصفحات، أو تقول "لاقي أي نمط شكله: 3 أرقام، شرطة، 3 أرقام، شرطة، 4 أرقام." Regex هي الطريقة الثانية — تصف النمط وهي تلاقي كل matches تلقائياً.

المكوّنات الأساسية

النوعماذا يفعلمثال
Metacharactersحروف ذات معنى خاص. بيطابق أي حرف، ^ بداية السطر
Special sequencesBackslash + حرف = اختصار\d = أي رقم، \w = أي حرف كلمة
Sets [ ]طابق أي حرف داخل الأقواس[aeiou] يطابق أي حرف علّة

الدوال الرئيسية في Python (مكتبة re)

  • findall() — بتلاقي كل الـ matches وتردّها كـ list
  • search() — بتلاقي أول match فقط
  • split() — بتقسّم النص عند كل match
  • sub() — بتبدّل الـ matches بنص جديد
مثال كود
import re text = "My email is john@example.com and work is john@corp.org" # إيجاد كل الإيميلات emails = re.findall(r'\b\w+@\w+\.\w+\b', text) # النتيجة: ['john@example.com', 'john@corp.org'] # إخفاء الإيميلات cleaned = re.sub(r'\b\w+@\w+\.\w+\b', '[HIDDEN]', text) # النتيجة: 'My email is [HIDDEN] and work is [HIDDEN]'
ملخص
Regex = أداة مطابقة الأنماط. في NLP بنستخدمها لتنظيف النص والـ tokenization. مكتبة re في Python فيها findall وsearch وsplit وsub.
03
Tokenization
▼

What is Tokenization?

Tokenization is splitting raw text into small units called tokens. It is a mandatory first step in all NLP pipelines — both traditional methods (like Count Vectorizer) and modern deep learning models (like Transformers) need it.

Analogy
Think of a sentence as a Lego set still in the box. Tokenization is the act of opening the box and separating each piece so you can work with them individually. You cannot build anything useful with a sealed box.

Types of tokens

TypeWhat it splits intoExample
Word tokensIndividual words"I love NLP" → ['I', 'love', 'NLP']
Sentence tokensIndividual sentencesParagraph → list of sentences
N-gram tokensGroups of N consecutive wordsBigram: "I love", "love NLP"
Regex-based tokensCustom patterns (e.g., keep apostrophes)"don't" stays as one token
Code Example (NLTK)
from nltk.tokenize import word_tokenize, sent_tokenize from nltk.util import ngrams text = "I love NLP. It is very powerful." # Word tokenization words = word_tokenize(text) # ['I', 'love', 'NLP', '.', 'It', 'is', 'very', 'powerful', '.'] # Sentence tokenization sents = sent_tokenize(text) # ['I love NLP.', 'It is very powerful.'] # Bigrams (N=2) bigrams = list(ngrams(words, 2)) # [('I','love'), ('love','NLP'), ('NLP','.'), ...]

Why N-grams?

Single words lose context. "New York" as two separate words loses the meaning. As a bigram "New York", the meaning is preserved. N-grams capture this kind of context.

Summary
Tokenization = splitting text into smaller units. It is mandatory in all NLP tasks. You can tokenize by words, sentences, or N-grams. Use NLTK's word_tokenize and sent_tokenize.

ما هو Tokenization؟

Tokenization هي عملية تقسيم النص الخام إلى وحدات صغيرة اسمها tokens. هي الخطوة الأولى الإلزامية في كل NLP pipeline — سواء الطرق التقليدية مثل Count Vectorizer أو الحديثة مثل Transformers.

تشبيه
فكّر في الجملة كصندوق Lego مغلق. الـ Tokenization هي فتح الصندوق وفصل كل قطعة لحالها حتى تقدر تشتغل عليها. ما قادر تبني شي مفيد من صندوق مغلق.

أنواع الـ Tokens

النوعماذا يقسّم إلىمثال
Word tokensكلمات منفردة"I love NLP" ← ['I', 'love', 'NLP']
Sentence tokensجمل منفردةفقرة ← list من الجمل
N-gram tokensمجموعات من N كلمات متتاليةBigram: "I love", "love NLP"
Regex-based tokensأنماط مخصصة"don't" تبقى كـ token واحد

ليش N-grams مهمة؟

الكلمات المنفردة بتفقد السياق. "New York" كـ كلمتين منفصلتين تفقد معناها. كـ bigram "New York" يُحفظ المعنى. الـ N-grams بتحتفظ بهذا النوع من السياق.

ملخص
Tokenization = تقسيم النص إلى وحدات أصغر. إلزامية في كل مهام NLP. ممكن تقسّم بكلمات أو جمل أو N-grams. استخدم word_tokenize وsent_tokenize من NLTK.
04
Removing Stop Words
▼

What are Stop Words?

Stop words are very common words that appear in almost every sentence but add little or no value for analysis. Examples: "a", "the", "is", "in", "and", "of".

Analogy
Imagine reading a book and trying to understand its topic. The words "the", "a", "and", "is" appear on every page but tell you nothing about what the book is about. If you removed them and kept only meaningful words like "dragon", "fire", "kingdom", you would understand the topic much faster. Stop word removal does exactly this.
Example

Original: "The cat is sitting on the mat in the house"

After removing stop words: "cat sitting mat house"

The meaning is still clear. The model now has fewer, more meaningful words to work with.

Important Note
Do NOT always remove stop words. For example, in sentiment analysis, words like "not" are stop words but they completely change meaning ("not good" vs "good"). Always think about your task first.
Code Example (NLTK)
from nltk.corpus import stopwords from nltk.tokenize import word_tokenize stop_words = set(stopwords.words('english')) text = "The cat is sitting on the mat" tokens = word_tokenize(text.lower()) filtered = [w for w in tokens if w not in stop_words] # Result: ['cat', 'sitting', 'mat']
Summary
Stop words are common, low-value words (the, a, is). Removing them reduces noise and speeds up models. But be careful — in some tasks like sentiment analysis, stop words like "not" matter a lot.

ما هي Stop Words؟

Stop Words هي كلمات شائعة جداً بتظهر في تقريباً كل جملة لكن ما بتضيف قيمة للتحليل. أمثلة: "a"، "the"، "is"، "in"، "and".

تشبيه
تخيّل إنك قاعد تقرأ كتاب وتحاول تفهم موضوعه. كلمات زي "the"، "a"، "and" موجودة في كل صفحة لكنها ما بتخبرك شي عن الموضوع. لو حذفتها وأبقيت بس الكلمات المهمة مثل "dragon"، "fire"، "kingdom"، ستفهم الموضوع بسرعة أكبر بكثير. هذا بالضبط ما يعمله حذف Stop Words.
مثال

الأصل: "The cat is sitting on the mat in the house"

بعد الحذف: "cat sitting mat house"

المعنى لا يزال واضحاً. الـ model الآن عنده كلمات أقل وأكثر معنى.

تحذير مهم
لا تحذف Stop Words دائماً. مثلاً في Sentiment Analysis، كلمة "not" هي stop word لكنها بتغيّر المعنى كلياً ("not good" مختلف عن "good"). فكّر دائماً في مهمتك أولاً.
ملخص
Stop Words = كلمات شائعة بقيمة منخفضة (the, a, is). حذفها بيقلّل الضوضاء ويسرّع الـ model. لكن انتبه — في بعض المهام مثل Sentiment Analysis، كلمات مثل "not" مهمة جداً.

Stemming
▼

What is Stemming?

Stemming is the process of cutting off the end of a word to get its root form (stem). The result is not always a real dictionary word — it is just the chopped version.

Analogy
Stemming is like a very aggressive hair cut — it just chops off whatever is at the end. Sometimes the result looks fine ("running" → "run"), sometimes it looks weird ("studies" → "studi"). It does not check if the result is a real word. It just cuts.
Examples
Original wordAfter stemmingReal word?
runningrunYes
studiesstudiNo
programmingprogramYes
happinesshappiNo

Stemmer algorithms in NLTK

  • Porter Stemmer — most common, moderate aggressiveness
  • Lancaster Stemmer — most aggressive, shortest stems
  • Snowball Stemmer — improved Porter, supports multiple languages
Code Example
from nltk.stem import PorterStemmer stemmer = PorterStemmer() words = ['running', 'studies', 'programming'] stems = [stemmer.stem(w) for w in words] # Result: ['run', 'studi', 'program']

When to use stemming?

Use when speed is a priority and you are okay with some inaccuracy — for example, in large-scale search or quick text classification tasks.

Summary
Stemming chops word endings to get the root. Fast but rough — results may not be real words. Use Porter (standard), Lancaster (aggressive), or Snowball (multilingual).

ما هو Stemming؟

Stemming هي عملية قطع نهاية الكلمة للوصول لجذرها (stem). النتيجة مش دائماً كلمة موجودة في القاموس — هي بس النسخة المقصوصة.

تشبيه
Stemming مثل قصة شعر عدوانية — بتقص أي شي في النهاية. أحياناً النتيجة منطقية ("running" ← "run")، وأحياناً تبدو غريبة ("studies" ← "studi"). ما بيتحقق إذا النتيجة كلمة حقيقية. بس يقص.
أمثلة
الكلمة الأصليةبعد Stemmingكلمة حقيقية؟
runningrunنعم
studiesstudiلا
programmingprogramنعم
happinesshappiلا

خوارزميات Stemming في NLTK

  • Porter Stemmer — الأكثر شيوعاً، عدوانية معتدلة
  • Lancaster Stemmer — الأكثر عدوانية، أقصر الجذور
  • Snowball Stemmer — تحسين لـ Porter، يدعم لغات متعددة

متى تستخدم Stemming؟

استخدمه لما السرعة أولوية وتقبل بعض الدقة المنخفضة — مثلاً في البحث على نطاق واسع أو تصنيف النصوص السريع.

ملخص
Stemming يقص نهايات الكلمات للوصول للجذر. سريع لكن تقريبي — النتائج قد لا تكون كلمات حقيقية. استخدم Porter (معياري)، أو Lancaster (عدواني)، أو Snowball (متعدد اللغات).

Lemmatization
▼

What is Lemmatization?

Lemmatization also reduces a word to its root, but unlike stemming, the result is always a real, valid dictionary word called a lemma. It uses a dictionary (WordNet) and understands the word's grammatical role (POS tag) to find the correct root.

Analogy
Think of lemmatization as a careful, educated editor. Instead of blindly chopping words, it first asks: "what part of speech is this word?" Then it looks up the proper root in the dictionary. "Was" → "be". "Better" → "good". These are intelligent, grammar-aware reductions.

Stemming vs Lemmatization — the key difference

FeatureStemmingLemmatization
OutputMay not be a real wordAlways a real dictionary word
MethodChop the ending (heuristic)Use a dictionary + POS tag
SpeedFasterSlower
AccuracyLowerHigher
Examplestudies → studistudies → study
Examplewas → wa (wrong)was → be (correct)
When to useSpeed is priorityQuality is priority
Code Example (NLTK WordNetLemmatizer)
from nltk.stem import WordNetLemmatizer lemmatizer = WordNetLemmatizer() # Without POS tag (defaults to noun) print(lemmatizer.lemmatize('running')) # running (wrong!) # With POS tag (verb) print(lemmatizer.lemmatize('running', pos='v')) # run (correct) print(lemmatizer.lemmatize('better', pos='a')) # good print(lemmatizer.lemmatize('studies', pos='v')) # study
Important
Always provide the POS tag when using WordNetLemmatizer. Without it, it defaults to treating the word as a noun, which often gives wrong results (e.g., "running" stays "running" instead of becoming "run").
Summary
Lemmatization is the smarter sibling of stemming. It always returns real dictionary words by using grammar context. Slower but more accurate. In modern NLP, it performs well without hurting speed much.

ما هو Lemmatization؟

Lemmatization كمان بتردّ الكلمة لجذرها، لكن بالعكس من Stemming، النتيجة دائماً كلمة حقيقية موجودة في القاموس اسمها lemma. بتستخدم قاموساً (WordNet) وبتفهم الدور النحوي للكلمة (POS tag) للوصول للجذر الصحيح.

تشبيه
فكّر في Lemmatization كمحرر متعلّم ودقيق. بدل ما يقص الكلمات بشكل أعمى، أولاً بيسأل: "ما هو دور هذه الكلمة نحوياً؟" ثم يبحث في القاموس عن الجذر الصحيح. "was" ← "be". "better" ← "good". هذه اختزالات ذكية تراعي قواعد اللغة.

POS Tagging (Part of Speech)
▼

What is POS Tagging?

Part of Speech (POS) Tagging is the process of labeling each word in a text with its grammatical role — noun, verb, adjective, adverb, etc. It is also called grammatical tagging.

Analogy
POS tagging is like color-coding a sentence. You highlight all nouns in blue, all verbs in green, all adjectives in yellow. After color-coding, both you and the machine can immediately see the structure of the sentence.
Example

Sentence: "The quick brown fox jumps over the lazy dog"

WordPOS TagMeaning
TheDTDeterminer
quickJJAdjective
brownJJAdjective
foxNNNoun (singular)
jumpsVBZVerb (3rd person singular)
lazyJJAdjective
dogNNNoun (singular)
Code Example (NLTK)
import nltk from nltk.tokenize import word_tokenize sentence = "The quick brown fox jumps" tokens = word_tokenize(sentence) tags = nltk.pos_tag(tokens) # [('The', 'DT'), ('quick', 'JJ'), ('brown', 'JJ'), ('fox', 'NN'), ('jumps', 'VBZ')]

Why does POS Tagging matter?

  • Required for correct lemmatization (need to know if a word is a verb or noun)
  • Useful in NER (Named Entity Recognition) — a name is usually a noun
  • Helps in parsing sentence structure
  • Important for disambiguation — "bat" can be an animal (noun) or to hit (verb)
Summary
POS Tagging = labeling every word with its grammatical role (NN, VB, JJ, etc.). It is essential input for lemmatization, NER, and many downstream NLP tasks.

ما هو POS Tagging؟

Part of Speech (POS) Tagging هي عملية وضع تصنيف نحوي على كل كلمة في النص — اسم، فعل، صفة، ظرف، إلخ. تُعرف أيضاً بـ grammatical tagging.

تشبيه
POS Tagging مثل تلوين الجملة. كل الأسماء باللون الأزرق، كل الأفعال بالأخضر، كل الصفات بالأصفر. بعد التلوين، أنت والماكينة تقدرون فوراً تشوفون هيكل الجملة.
مثال

الجملة: "The quick brown fox jumps over the lazy dog"

الكلمةPOS Tagالمعنى
quickJJصفة (Adjective)
foxNNاسم مفرد (Noun)
jumpsVBZفعل (Verb)
lazyJJصفة (Adjective)

ليش POS Tagging مهم؟

  • مطلوب للـ lemmatization الصحيح (لازم تعرف هل الكلمة فعل أو اسم)
  • مفيد في NER — الاسم عادةً يكون noun
  • يساعد في فهم هيكل الجملة
  • مهم لفك الغموض — كلمة "bat" ممكن تكون حيوان (اسم) أو فعل ضرب
ملخص
POS Tagging = وضع تصنيف نحوي على كل كلمة (NN، VB، JJ، إلخ). ضروري للـ lemmatization والـ NER وكثير من مهام NLP الأخرى.
08
Automatic Tagging
▼

What is Automatic Tagging?

Instead of manually labeling every word, we can train a system to automatically assign POS tags. This is done using annotated corpora — large collections of pre-labeled text.

What is a Corpus?

A corpus (plural: corpora) is a large collection of text. The Brown Corpus was the first million-word electronic corpus of English, created in 1961. It contains 500 sources (news, government, editorial, etc.) all manually tagged.

Types of Automatic Taggers

TaggerHow it worksAccuracy
Default TaggerAssigns the same tag to every word (most common tag — usually NN). Baseline only.~13%
Regex TaggerAssigns tags based on word patterns. Words ending in "-ed" → past tense verb.Medium
Unigram TaggerFor each word, assigns the most common tag it had in training data.~93%
Bigram TaggerUses current word + previous tag as context.Higher
Trigram TaggerUses current word + 2 previous tags as context.Highest
Brill's TaggerStarts with initial guess, then applies correction rules to fix mistakes. Rule-based.High
Analogy — Combining Taggers
Think of the fallback strategy like asking experts in order: "First, ask the specialist (bigram tagger). If they don't know, ask the general doctor (unigram tagger). If they don't know either, use the default answer." This is called a backoff strategy.

Brill's Tagger — the unique approach

Brill's tagger is different. It starts by guessing all tags, then applies transformation rules to fix mistakes — like "if a word is tagged as Noun but the next word is a Verb, change it to Adjective." It compiles a list of correction rules from training data.

Key ML concept: Train/Test Split
When training a tagger, never test it on the same data you trained it on. Use 90% for training and 10% for testing. A tagger that memorizes training data gets 100% accuracy on it but fails on new text — this is called overfitting.
Summary
Automatic tagging learns POS tags from labeled corpora. Accuracy improves as we add more context: Default → Regex → Unigram → Bigram → Trigram. Combining taggers with backoff is a practical, accurate approach.

ما هو Automatic Tagging؟

بدل ما نصنّف كل كلمة يدوياً، ممكن نتدرّب نظاماً ليضع POS tags تلقائياً. هذا يتم باستخدام corpora مصنّفة مسبقاً — مجموعات نصوص كبيرة ومعلّمة.

ما هو الـ Corpus؟

الـ corpus (الجمع: corpora) هو مجموعة كبيرة من النصوص. الـ Brown Corpus كان أول corpus إلكتروني بمليون كلمة إنجليزية، أُنشئ سنة 1961. يحتوي على 500 مصدر (أخبار، حكومة، مقالات، إلخ) مصنّفة يدوياً.

أنواع الـ Taggers التلقائية

النوعكيف يعملالدقة
Default Taggerيعطي نفس الـ tag لكل كلمة (الأكثر شيوعاً — عادةً NN). فقط للقياس الأساسي.~13%
Regex Taggerيعطي tags بناءً على أنماط الكلمة. الكلمات المنتهية بـ "-ed" → فعل ماضي.متوسطة
Unigram Taggerلكل كلمة يعطي الـ tag الأكثر شيوعاً من بيانات التدريب.~93%
Bigram Taggerيستخدم الكلمة الحالية + الـ tag السابق كسياق.أعلى
Trigram Taggerيستخدم الكلمة الحالية + 2 tags سابقة كسياق.الأعلى
Brill's Taggerيبدأ بتخمين الـ tags، ثم يطبّق قواعد تصحيح لإصلاح الأخطاء.عالية
تشبيه — استراتيجية الاحتياط (Backoff)
فكّر فيها مثل سؤال الخبراء بالترتيب: "أولاً اسأل الأخصائي (bigram tagger). إذا ما عرف، اسأل الطبيب العام (unigram tagger). إذا ما عرف كمان، استخدم الإجابة الافتراضية." هذا اسمه backoff strategy.
مفهوم ML مهم: Train/Test Split
لما تتدرّب على tagger، لا تختبره على نفس البيانات اللي تدرّبت عليها. استخدم 90% للتدريب و10% للاختبار. الـ model اللي يحفظ بيانات التدريب بيطلع دقة 100% عليها لكن يفشل مع نصوص جديدة — هذا اسمه overfitting.
ملخص
Automatic Tagging بيتعلم الـ POS tags من corpora مصنّفة. الدقة بتتحسن كلما زاد السياق: Default ← Regex ← Unigram ← Bigram ← Trigram. الجمع بين الـ taggers مع backoff نهج عملي ودقيق.

What is NER?

Named Entity Recognition (NER) is the task of finding and classifying named entities in text into predefined categories like:

  • Person names (Mohammed, Elon Musk)
  • Organizations (Google, WHO)
  • Locations (Riyadh, Egypt)
  • Dates and times (January 2024, 3 PM)
  • Monetary values ($500, 100 SAR)
  • Percentages (20%)
Analogy
Think of NER like a highlighter that reads a news article. It uses a blue highlighter for all people's names, a green one for all companies, and a yellow one for all locations. After NER, the article is color-coded and you instantly know who, where, and what is being talked about.
Example

Input: "Apple was founded by Steve Jobs in California in 1976."

Steve JobsPERSON
CaliforniaGPE (Location)
1976DATE

NER is a subtask of Information Extraction

NER is used to extract structured information from unstructured text. For example, from thousands of news articles, you can extract all mentioned companies, people, and locations automatically.

Summary
NER = identifying and classifying named entities (people, places, organizations, dates) in text. It is a critical tool for information extraction from unstructured data.

ما هو NER؟

Named Entity Recognition (NER) هي مهمة إيجاد وتصنيف الـ entities المسمّاة في النص إلى فئات محددة مسبقاً مثل:

  • أسماء الأشخاص (محمد، إيلون ماسك)
  • المنظمات (Google، WHO)
  • المواقع (الرياض، مصر)
  • التواريخ والأوقات (يناير 2024، الساعة 3)
  • القيم المالية ($500)
  • النسب المئوية (20%)
تشبيه
فكّر في NER كـ highlighter بيقرأ مقال إخباري. يستخدم highlighter أزرق لأسماء الأشخاص، أخضر للشركات، وأصفر للمواقع. بعد NER، المقال ملوّن وتعرف فوراً عمَن ومن أين وماذا يتحدث.
مثال

الإدخال: "Apple was founded by Steve Jobs in California in 1976."

NER هي جزء من Information Extraction

NER بتُستخدم لاستخراج معلومات منظّمة من نص غير منظّم. مثلاً، من آلاف المقالات الإخبارية تقدر تستخرج تلقائياً كل الشركات والأشخاص والأماكن المذكورة.

ملخص
NER = تحديد وتصنيف الـ entities المسمّاة (أشخاص، أماكن، منظمات، تواريخ) في النص. أداة أساسية لاستخراج المعلومات من البيانات غير المنظّمة.
